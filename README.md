# YourVPN — Telegram Mini App

Минималистичная выдача персонального VPN-ключа для вставки в Happ (или
любой другой VLESS/Xray-совместимый клиент). Без кнопки «Подключиться» —
само подключение происходит в стороннем приложении, здесь только выдача
и удобное копирование ключа.

## Как это подключить как настоящий Mini App

Telegram Mini App — это просто веб-страница (этот `index.html`), которую
Telegram открывает внутри чата/меню бота. Нужно:

1. **Захостить `index.html`** на любом HTTPS-хостинге (обязательно HTTPS —
   Telegram не откроет `http://`). Подойдёт Netlify, Vercel, Cloudflare
   Pages — у всех есть бесплатный тариф, достаточный для этого.
2. **Создать/открыть бота через [@BotFather](https://t.me/BotFather)**:
   - `/mybots` → выбрать бота → **Bot Settings** → **Menu Button**
   - Указать URL вашего захостенного `index.html`
   - Либо через `/newapp`, если хотите отдельное Mini App с собственной
     иконкой и названием в каталоге приложений Telegram.
3. Всё — кнопка меню бота (или сама команда `/start`) будет открывать это
   приложение прямо внутри Telegram.

## Что уже готово в коде

- **Автоматическая тема** — приложение подстраивается под тему пользователя
  в Telegram (тёмная/светлая, акцентные цвета) через `Telegram.WebApp.themeParams`.
- **Нативная нижняя кнопка** — использует `MainButton` из Telegram SDK
  (выглядит как родной элемент интерфейса, а не веб-кнопка), с фолбэком на
  свою кнопку, если открыть страницу не в Telegram (для превью/тестов).
- **Копирование ключа** — через Clipboard API с фолбэком для старых WebView.
- **QR-код** — генерируется через публичный API по клику, для тех, кто
  предпочитает сканировать в Happ, а не вставлять текст.
- **Тактильная отдача (haptics)** — при копировании и обновлении ключа.

## Что нужно заменить на реальное

Всё промаркировано в коде комментарием `MOCK KEY SOURCE`:

```js
function mockFetchKey() {
  const fakeUUID = 'demo-' + Math.random().toString(16).slice(2, 10) + '-xxxx';
  return `vless://${fakeUUID}@de.yourvpn.com:443?type=tcp&security=reality&sni=example.com`;
}
```

В продакшене вместо генерации фейкового ключа на клиенте нужно:

1. Отправить на свой backend `tg.initData` (это подписанные Telegram данные
   о пользователе — забирает Mini App автоматически, ничего вводить не надо).
2. **На backend обязательно проверить подпись `initData`** (Telegram даёт
   для этого секрет вашего бота) — это подтверждает, что запрос
   действительно пришёл от Telegram, а не подделан.
3. По проверенному Telegram ID пользователя backend возвращает **его
   персональный** ключ (создать заранее в Marzban/своей VPN-панели, либо
   генерировать при первом обращении через API панели).
4. Подставить полученный ключ в `renderKey(key)` вместо мока.

## Про Happ

Сейчас ссылки на App Store / Google Play — это заглушки-плейсхолдеры
(`happIosLink` / `happAndroidLink` в коде). Пришлите точные ссылки на Happ
в сторах, и, если у Happ есть свой URL-scheme для приёма ключа напрямую
(например, что-то вроде `happ://add?url=...`), я сделаю кнопку
«Добавить одним тапом» вместо шага «скопировать → вставить вручную» —
это сильно упростит подключение для пользователей.

## Вибрация при успешной оплате

При успехе оплаты — мягкая двухфазная вибрация (короткий тап → пауза →
более длинный buzz), в духе уведомления о письме, а не резкий одиночный
щелчок. Технически это не "фирменный паттерн Gmail" дословно — Google
нигде не публикует точные тайминги как спецификацию, так что это наша
собственная интерпретация в том же стиле.

Как это работает по платформам:
- **Android / браузеры с поддержкой `navigator.vibrate()`** — используется
  настоящий кастомный паттерн `[45, 90, 110]` (мс: буз-пауза-буз).
- **iOS / внутри Telegram** — у `Telegram.WebApp.HapticFeedback` нет API
  для произвольных таймингов, только готовые типы. Имитируем тот же
  двухфазный эффект последовательностью `impactOccurred('soft')` →
  (через 220мс) → `notificationOccurred('success')`.

Сама анимация успеха тоже стала менее резкой: обработка платежа теперь
идёт ~2.6 секунды (было ~1.7) — не спешит, ощущается солиднее — а
зелёная галочка появляется мягче и после появления слегка "дышит"
пульсирующим свечением около 2 секунд, а не просто статично замирает.

## Оплата — что мок, а что нужно подключить по-настоящему

Три способа оплаты в интерфейсе, у каждого своя степень готовности:

### 💳 Банковская карта
Сейчас — просто анимация "обработка → успех". **Важно:** ввод номера карты
никогда не должен собираться собственным кодом сайта/приложения — это
требует сертификации PCI DSS, которой ни у одного самостоятельного
разработчика нет. Правильная схема: кнопка "Оплатить картой" ведёт на
**хостинг-страницу оплаты платёжного шлюза** (YooKassa, CloudPayments,
Т-Касса — все работают с российскими картами) — сами данные карты вводятся
там, на их защищённой инфраструктуре, а не в вашем коде. Как только
зарегистрируете ИП и подключите один из этих шлюзов — пришлёте API-ключ,
и я переключу `startPayment()` на реальный редирект.

### 📱 СБП по QR
QR-код сейчас — просто визуальная заглушка. Настоящий QR для СБП
генерируется тем же платёжным шлюзом на лету под каждую транзакцию
(с суммой, получателем и сроком действия) — подключается одновременно с
картой, через тот же шлюз.

### ₿ Криптовалюта
Адрес, который сейчас показывается под QR — **явно помечен как ПРИМЕР**,
это не настоящий кошелёк (я не могу и не должен придумывать вам чужие
платёжные реквизиты). Для реальной работы: либо подставьте свой
собственный адрес, либо (надёжнее) подключите крипто-платёжный процессор
(CryptoCloud, Plisio и подобные) — они генерируют уникальный адрес на
каждый платёж и сами подтверждают поступление, что избавляет от ручной
сверки платежей.

## Реферальная программа

Ссылка вида `https://t.me/ваш_бот?start=ref_<telegram_id>` формируется
автоматически на основе ID текущего пользователя Telegram. Награда сейчас
задана как **+7 дней Premium** за каждого приглашённого, кто оформит
подписку — эту цифру легко поменять в HTML (`.ref-sub`).

**Важно:** количество приглашённых и начисленные дни сейчас отображаются
как `0` (мок) — это не хранится нигде, при перезагрузке сбрасывается.
Для настоящей реферальной программы нужен backend:

1. Ваш Telegram-бот обрабатывает `/start ref_<id>` на сервере — именно
   так Telegram передаёт, кто по чьей ссылке пришёл.
2. Backend сохраняет связь "пригласивший → приглашённый" в базе.
3. Когда приглашённый оплачивает подписку — backend начисляет бонусные
   дни обоим, **не полагаясь на данные с клиента** (иначе пользователь
   сможет накрутить себе дни, подделав запрос).
4. Mini App запрашивает актуальную статистику у backend при открытии
   (`GET /miniapp/referrals`) вместо захардкоженных нулей.
